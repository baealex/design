Invisible Interface

Mono

Content is the only color.
Everything else whispers.

Used in BLEX

Monochrome strips the interface down to black, white, and gray. No accent colors. No brand tints. Content — text, images, code — provides all the color the UI needs.

The chrome disappears, leaving only what matters. Where other systems add color to create hierarchy, Monochrome uses weight, spacing, and shades of gray to make the interface invisible and the content vivid.

Principle 01

Radius as Intent

Border radius isn't decoration — it's a semantic signal. Larger radii hold more content. Smaller radii demand faster action. The radius tells you what the element does before you read a word.

Principle 03

Motion Is Meaning

Every transition exists for a reason. 150ms for interactions — fast enough to feel instant, slow enough to register. 500ms for entrance — just enough time to orient. Apple's easing curve for everything.

/* Monochrome — motion specs */
--duration-interaction: 150ms;
--duration-entrance: 500ms;
--easing: cubic-bezier(0.25, 0.1, 0.25, 1);

/* Touch feedback */
active: scale(0.98);

/* Shadow hierarchy */
--shadow-sm: 0 1px 3px rgba(0,0,0,0.04);
--shadow-md: 0 4px 12px rgba(0,0,0,0.08);
--shadow-lg: 0 8px 24px rgba(0,0,0,0.12);
Principle 04

Glass for What Floats

Elements that live inside content are solid white. Elements that float above content are glass — backdrop-blur with semi-transparent white. The blur tells you it's hovering. The transparency keeps you grounded.

white/80 · blur(12px)
Content sits behind the glass. The header blurs it gently, just enough to separate layers without hiding anything. The floating action bar does the same — pill-shaped, translucent, casting a soft shadow. Glass for what floats. Solid for what sits. That's the rule.

All Together

White Surface, Gray System

A card that needs no color to communicate. Title weight pulls the eye. Gray text steps back. Subtle borders contain. The content — whatever it may be — is the only thing that gets to be vivid.

What This Is Not

  • ✗ Gray because you're lazy — this gray has a system

  • ✗ Adding "just a little blue" for links — resist
  • ✗ Flat design — we have shadows, just subtle ones
  • ✓ Content provides all the color
  • ✓ Every gray has a semantic role
  • ✓ Invisible chrome, visible content
  • ✓ Glass for overlays, solid for content
  • ✓ Radius and weight carry the hierarchy
