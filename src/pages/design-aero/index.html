Inspired by Windows Aero UI

Aero

Vista put glass in your OS.
We're putting purpose in the glass.

Windows Aero was a GPU flex. Translucent title bars, glossy buttons catching imaginary light, shadows that cost actual rendering budget. Everyone called it eye candy. That was fair — it mostly was.

But underneath the vanity? Some genuinely brilliant ideas about light, depth, and surface. We're keeping those.

Principle 01

Glass That Means Something

backdrop-filter: blur(). A few CSS properties and you've got a glass panel. The original Aero used DWM compositing that ate your battery for breakfast. We do it in CSS. The trick is knowing when to stop — too little blur and it's noisy, too much and you've painted a wall.

Principle 02

The Split Highlight

This is the trick. A linear-gradient with four stops, a hard break at 50%. Top half bright, bottom half dim. It mimics how light hits a convex glass surface. Microsoft's designers figured this out in 2006. It still looks good. Not bad for a technique older than the iPhone.

/* The Aero glossy gradient — 4 stops, hard break at 50% */
background: linear-gradient(180deg,
  rgba(255,255,255,0.35) 0%,   /* top — bright highlight */
  rgba(255,255,255,0.08) 50%,  /* middle — fade out */
  rgba(255,255,255,0.02) 51%,  /* hard break — shadow line */
  rgba(255,255,255,0.12) 100%  /* bottom — subtle fill */
);
Principle 03

Light Always Comes From Above

Every glass panel on this page has a secret: a pseudo-element gradient covering the top half, fading from white to transparent. Same trick photographers use — top-light makes things look three-dimensional. Remove it and the panels look dead. We tested.

What This Is Not

  • ✗ Making your GPU cry for a taskbar

  • ✓ Blur as visual hierarchy — foreground over background
  • ✓ Light simulation with pure CSS gradients
  • ✓ Performance-conscious translucency
