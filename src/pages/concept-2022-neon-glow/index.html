Neon Glow treats UI elements as light sources, not surfaces. Where neumorphism extruded plastic and glassmorphism frosted glass, Neon Glow makes elements glow like neon tubes in a dark room.

One color family. One darkness. The glow itself carries all the hierarchy you need — dim for rest, bright for action, blinding for urgency.

Principle 01

Glow Intensity Is Hierarchy

The same purple at three brightness levels tells three different stories. Dim is background, lit is interactive, bright is urgent. No second color needed.

Principle 02

Darkness Is the Canvas

Glow on a white background is just colored text. Glow on darkness is a light source. Same element, completely different physics. The darkness isn't absence — it's what makes the glow real.

Principle 03

One Color. All the Hierarchy.

The temptation is to add pink for warm, blue for cool, green for success. Resist. A single color family at different intensities creates a more cohesive, more atmospheric system than any rainbow.

What This Is Not

  • ✗ text-shadow: 0 0 99px on everything
  • ✗ Rainbow RGB on every surface — one color family only
  • ✗ Glow without darkness to justify it
  • ✗ Cheap cyberpunk cosplay — all grime, no system
  • ✓ Glow that simulates actual light emission
  • ✓ Intensity as the only hierarchy dial
  • ✓ Pitch darkness as a deliberate design surface
  • ✓ One color — the constraint is the power
