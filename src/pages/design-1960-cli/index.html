Inspired by CLI & TUI

CLI/TUI

No interface ever packed more power per pixel.
We kept the density and the speed. We lost the learning curve.

The Command Line Interface was the most efficient interface ever built. A Teletype Model 33 printed READY in 1964, and for three decades that blinking prompt was the interface — monospace grids, input-output loops, raw information density that no GUI has ever matched.

But it demanded memorization. It punished typos. It assumed the user already knew everything. That was the weakness. The density, the speed, the directness — those were the strengths, and they never went away. Every Cmd+K palette, every AI chat, every terminal emulator proves it. This page keeps what the CLI got right and builds it into a usable, modern interface that doesn't require a manual to operate.

Principle 01

The Monospace Grid

Every character occupies the same width. That single constraint creates perfect vertical alignment without CSS Grid, without Flexbox, without any layout engine at all. Columns line up because the font forces them to. Tables, borders, charts — all drawn with text alone. This is the original design system: 80 columns x 24 rows.

Principle 02

Command & Response

You type. The system answers. No loading spinners, no skeleton screens, no optimistic updates. The input-output loop is the most honest interaction pattern ever designed. It's also the ancestor of every chatbot, every AI assistant, every command palette you use today. Cmd+K is just a terminal that went to design school.

guest@design-system ~
Welcome to Design System v1.0
Type a command or click one below.
 
$
Principle 03

Information Density

A terminal shows more data per pixel than any GUI ever built. No padding, no whitespace worship, no card-based layouts that wrap three words in a 200px box. Just raw information, structured by alignment and box-drawing characters. This is what htop understood: density is not clutter when the structure is clear.
